DEMO MODE (ephemeral — a pod restart drops all state).

Verify the whole stack (helm test waits for readiness itself; add --wait to the
install above if you prefer the release to block until pods are Ready):
  helm test {{ .Release.Name }}

Reach MCP (keep it cluster-internal — the demo issuer mints valid kdive tokens for
any caller; never expose it via NodePort/LoadBalancer/Ingress):
  kubectl port-forward svc/{{ include "kdive.fullname" . }}-server 8000:8000

Mint a demo token for your own client (expires per the issuer's tokenExpiry). The token
carries the demo RBAC grant in demo.oidc.claims (default: admin on project `demo` + all
three platform roles), so it can drive the whole authz surface; narrow it with
`--set demo.oidc.claims...` to test a denial. Mint from inside a running pod (the demo
NetworkPolicy blocks ad-hoc pods from the issuer, and the token's `iss` must be the
in-cluster issuer URL the server validates against, not a port-forwarded localhost one):
  TOKEN=$(kubectl exec deploy/{{ include "kdive.fullname" . }}-server -- python -c \
    'import json,urllib.request as u; d=b"grant_type=client_credentials&client_id=kdive-demo&client_secret=x"; print(json.load(u.urlopen(u.Request("http://{{ include "kdive.fullname" . }}-oidc:8080/default/token",data=d),timeout=10))["access_token"])')
From a source checkout the same mint is wrapped in a helper:
  export KDIVE_TOKEN=$(scripts/demo-token.sh)     # KDIVE_DEMO_{NAMESPACE,FULLNAME,CONTEXT} override
To demonstrate an RBAC denial, mint a narrowed token (project role only, no platform roles):
  export KDIVE_TOKEN=$(scripts/demo-token.sh --role viewer)   # or: --role operator
Then drive http://127.0.0.1:8000/mcp with an MCP client and `Authorization: Bearer $TOKEN`
(the streamable-HTTP transport needs the MCP initialize handshake — a raw tools/list POST 307s).

# M1.3 — Platform Operations (Integration Contract)

## Purpose

M1.3 builds the **mutating platform-operations surface** that M1.1 deferred. M1.1 settled the
platform-role model (`platform_admin` / `platform_operator` / `platform_auditor`), the
`require_platform_role` seam, and the `platform_audit_log` read-access table, but shipped only
one consumer (`accounting.report`). [ADR-0043](../adr/0043-platform-scoped-rbac-tier.md) §5
deferred every *mutating* platform tool and the second/third auditor reads to M1.3 so they
could be shaped against operational use. M1.3 ships them. **Because the authorization machinery
already exists, M1.3 adds tools and one hardening retrofit — not authorization plumbing.**

## What M1.3 adds to M1.1

- **`platform_operator` infra tools** — `resources.set_status`, `resources.cordon` /
  `uncordon`, `resources.drain` (`mode=passive`/`force_release`); `ops.reconcile_now`;
  `ops.queue_pause` / `queue_resume` / `jobs_list`; `ops.set_cost_class_coeff` /
  `set_host_capacity`.
- **`platform_admin` break-glass** — `ops.force_teardown`, `ops.force_release` — a separate
  path that fully overrides the three-check destructive gate, with a mandatory `reason` and an
  always-written `platform_audit_log` row.
- **The remaining `platform_auditor` reads** — `audit.query` (project `admin` form +
  cross-project `platform_auditor` form) and `inventory.list` (cross-project systems/allocations
  summary).
- **The bare-`require_role` denial-audit retrofit** — a centralized dispatch-boundary catch
  writing `transition='denied'` rows to `audit_log` via a guard-exempt `audit.record_denial()`.

## Non-goals (scoped out)

- **No job requeue/force-fail or worker orchestration** — queue control is pause/resume +
  cross-project inspect only; the reconciler already auto-repairs abandoned jobs (ADR-0062 §1).
- **No runtime `W_CPU`/`W_MEM` weight tuning** — the global reference weights stay code-config;
  only cost-class coefficients and per-host capacity caps are runtime knobs.
- **No per-project budget/quota change** — those stay on `accounting.set_budget` /
  `accounting.set_quota` (project `admin`).
- **No persisted `draining` status** — drain is an action over the `cordoned` state (ADR-0062 §3).
- **No new role model** — `platform_roles`, `require_platform_role`, `platform_audit_log` are
  M1.1; M1.3 only consumes them. No project/role management (the IdP issues claims).
- **No core/plane interface change** — M1.3 is tools over settled seams.

## Postgres schema (M1.3 delta)

- **`resources.status`** gains the value **`cordoned`** — a healthy, admission-skipped host
  (distinct from `offline`).
- **`ops_control`** — a new single-row table: `queue_paused boolean NOT NULL DEFAULT false`
  (with `updated_at` and a single-row guard). The worker reads it before each `dequeue`.
- **`audit_log`** — `project` / `object_kind` / `object_id` become **nullable**, guarded by a
  CHECK requiring them for every transition except `'denied'`; a `reason text` column is added
  (set by break-glass and denial rows, NULL elsewhere).

No new table for pricing/capacity: cost-class coefficients (`cost_class_coefficients`) and host
caps (resource `capabilities` jsonb) are already DB-backed.

## MCP tool surface (M1.3 delta)

```
Resources (host ops — platform_operator)
  resources.set_status(resource_id, status)         status ∈ {available, degraded, offline}
  resources.cordon(resource_id)                      status → cordoned (admission skips)
  resources.uncordon(resource_id)                    cordoned → available
  resources.drain(resource_id, mode=passive|force_release)
                                                     cordon; passive reports live allocations,
                                                     force_release force-releases each (reason)

Ops (control plane — platform_operator)
  ops.reconcile_now()                                one reconcile_once() pass; returns summary
  ops.queue_pause() / ops.queue_resume()             toggle queue_paused
  ops.jobs_list(filters…)                            cross-project queue depth / job states
  ops.set_cost_class_coeff(cost_class, coeff)        upsert pricing coefficient
  ops.set_host_capacity(resource_id, concurrent_allocation_cap)

Ops (break-glass — platform_admin)
  ops.force_teardown(system_id, reason)              bypasses the three-check gate; always audited
  ops.force_release(allocation_id, reason)           bypasses the three-check gate; always audited

Audit / inventory (reads)
  audit.query(filters…)   project form  → require_role(project, admin)
                          x-project form → platform_auditor [read-audited]
  inventory.list(filters…)               → platform_auditor [read-audited]
```

Unchanged: per-project `systems.teardown` / `allocations.release` / `jobs.*` / `resources.list`
/ `.describe` / `accounting.set_budget` / `.set_quota`.

## Auth / RBAC delta

All new tools gate on the M1.1 `require_platform_role` seam: `platform_operator` for infra and
control-plane ops, `platform_admin` for break-glass, `platform_auditor` for the cross-project
reads (satisfied by `platform_admin` per ADR-0043 §2). `audit.query`'s project-scoped form rides
`require_role(project, admin)`. Break-glass **fully overrides** the three-check destructive gate
(authority = `platform_admin` + mandatory `reason` + audit row). Cross-tenant actions record to
`platform_audit_log`; `require_role` denials record to `audit_log` (`transition='denied'`). The
deployment root remains out-of-band (ADR-0043 §7).

## Exit criteria

Falsifiable; each becomes a test. (Full statements in the detailed design §Exit criteria.)

1. **Cordon** excludes a host from placement without touching its existing allocations;
   `uncordon` restores it; the `set_status`/`cordon` verbs cannot cross axes.
2. **Drain** `mode=passive` leaves live allocations running; `mode=force_release` empties the
   host; both leave it `cordoned`.
3. **`reconcile_now`** resolves a pending repair in one call and returns a summary, leaving the
   periodic loop unaffected.
4. **Queue pause** stops new claims but not a job already in flight; `resume` restores claiming.
5. **Tuning** — `set_cost_class_coeff` changes the next charge (not committed ledger rows);
   `set_host_capacity` changes admission's cap without evicting.
6. **Break-glass** succeeds against a cross-project object that fails the three-check gate for a
   non-member `platform_admin`; a blank `reason` is rejected; every call writes a
   `platform_audit_log` row; a `platform_operator` token is denied.
7. **Auditor reads** — `audit.query` project form requires `admin` on that project; its
   cross-project form and `inventory.list` require `platform_auditor`, are denied to a
   project-only token, and write a `platform_audit_log` row (not an `audit_log` row).
8. **Denial-audit** — a denied per-project call writes one `audit_log` row with
   `transition='denied'` (principal/agent_session/tool, object NULL when unresolvable); the
   CHECK rejects a non-`denied` row with a NULL object; the success path is unchanged.

# M1.5 ‚Äî Fault-injection provider (Integration Contract)

## Purpose

M1.5 adds a **mock provider behind the real plane interfaces** that **forces secret
resolution** and **injects seeded, reproducible latency and failures**, so the
reconciliation/teardown passes, the secret-registration‚Üíredaction contract, the
admission-control concurrency locks, and the cancel/compensation policies are all exercised
end-to-end by a provider that can be driven to **any** drift state on demand ‚Äî **before** any
real remote provider (M2) makes the same paths a production incident. It is the last
local-band milestone before the provider-expansion milestones; it ships the **per-kind
provider selection seam** M2 will reuse, but **no new agent-facing tools** ‚Äî the fault-inject
resource is driven through the existing `allocations.*` / `systems.*` / `runs.*` / `debug.*` /
`control.*` surface.

- **Decisions:** [ADR-0071](../adr/0071-per-kind-provider-runtime-registry.md) (per-kind
  `ProviderRuntime` registry ‚Äî the selection seam), [ADR-0072](../adr/0072-fault-injection-provider-seeded-engine.md)
  (the fault-inject provider + the seeded **decision-keyed** fault engine),
  [ADR-0073](../adr/0073-forced-secret-resolution-redaction.md) (forced secret resolution +
  end-to-end redaction validation). All build on
  [ADR-0063](../adr/0063-typed-provider-runtime.md) (the typed `ProviderRuntime` seam they
  generalize), [ADR-0021](../adr/0021-reconciler-loop-drift-repair.md) (the reconciler passes
  the provider triggers), [ADR-0036](../adr/0036-reservation-lease-semantics.md) (the lease a
  slow provision races), [ADR-0040](../adr/0040-admission-lifecycle-concurrency.md) (the
  per-project / per-resource locks the race tests contend), and
  [ADR-0027](../adr/0027-safety-modules-secret-backend-impl.md) / [ADR-0012](../adr/0012-secret-backend.md)
  (the register-before-return `SecretBackend`).
- **Parent:** [`top-level-design.md`](top-level-design.md) ¬ßRoadmap (M1.5).

## What M1.5 adds

- **A per-kind `ProviderRuntime` registry (the selection seam)** ‚Äî the single global
  `build_default_provider_runtime()` becomes a `ResourceKind ‚Üí ProviderRuntime` map in
  `providers/composition.py`; **post-System** ops at the worker and MCP boundaries resolve the
  runtime from the **System's Resource `kind`** (the pre-grant allocation plane and discovery
  do **not** key on a Resource ‚Äî see ¬ßProvider model), not a process constant (ADR-0071). A
  new `ResourceKind.FAULT_INJECT = "fault-inject"` and migration `0018` (CHECK widen) register
  the second kind. local-libvirt's resolved runtime is **byte-identical** to today's ‚Äî a
  **checkable** claim split into a CI-enforceable gate (issue 1 diffs no behavior under
  `providers/local_libvirt/*`; CI tests untouched and green) and an operator-run gate (the
  M1.2 e2e suite passes out of band) ‚Äî so M1.5 changes the *selection*, not the *behavior*, of
  the shipped provider. An unknown kind is a `configuration_error`, never a silent fallthrough.
  This is the seam M2 plugs into; it lives in the provider composition module (the expected
  change-surface), so building it does not falsify the top-level design's "adding a provider
  touches zero `core/*` lines" hypothesis ‚Äî it makes that hypothesis testable against a real
  second provider for the first time.
- **A fault-injection provider** ‚Äî a mock satisfying every typed port
  (`Provisioner`/`Builder`/`Installer`/`Booter`/`Connector`/`Controller`/`Retriever`/debug/
  introspection) with synthetic-but-plausible outputs: a synthetic domain name from
  `provision`, a loopback `TransportHandle` from `connect`, a synthetic vmcore artifact from
  `retrieve` ‚Äî so the **happy path** (no drawn fault) drives the full spine
  (admission ‚Üí job ‚Üí worker ‚Üí reconciler) to a real terminal state. It owns a **mock
  infra-inventory seam** (`list_owned`/`destroy`, the `InfraReaper` shape) so the reconciler's
  leaked-domain pass has synthetic infra to find and reap (ADR-0072).
- **A seeded, decision-keyed fault engine** ‚Äî each fault decision is a **pure function of
  stable inputs**: `fault_for(seed, system_id, plane, attempt, facet) ‚Üí draw in [0,1)`, where
  `facet` ‚àà {`fail`, `category`, `latency`} keys three independent draws ‚Äî **not** a step of a
  shared mutable PRNG stream (order-dependent under concurrent workers, flaky even with a fixed
  seed). Three details are load-bearing (ADR-0072): the draw uses a **process-independent**
  hash (`hashlib.blake2b`/`sha256`, **never** builtin `hash()`, which `PYTHONHASHSEED`-salts
  strings per process); **`attempt` derives from durable state** (Run boot ordinal / attach
  ordinal / persisted retry count), never a process-local counter; and the `latency` draw
  scales against a per-plane **`max_latency_s`** so a delay can outlast a short lease. The
  `fail` draw is compared against the plane's `fault_rate` (a **per-plane map**, not a scalar);
  the `category` draw buckets among that plane's `ErrorCategory` options. `seed`, the
  `fault_rate` map, `max_latency_s`, and `secret_ref` ride the `capabilities` jsonb, set by
  discovery, **never** read from wall-clock or `os.urandom` (a guard test asserts the draw is
  identical across two subprocesses with different `PYTHONHASHSEED`). The catalog maps only to
  **existing** `ErrorCategory` values (`provision`‚Üí`PROVISIONING_FAILURE`,
  `install`‚Üí`INSTALL_FAILURE`/`BOOT_TIMEOUT`, `boot`‚Üí`READINESS_FAILURE`/`BOOT_TIMEOUT`,
  `connect`‚Üí`TRANSPORT_FAILURE`, `control`‚Üí`CONTROL_FAILURE`, `retrieve`‚Üí`INFRASTRUCTURE_FAILURE`).
  A test sets a plane's `fault_rate` high and pins a seed to fail it (assertable); a soak run
  sweeps seeds (broad) ‚Äî the same engine at two settings (ADR-0072).
- **Forced secret resolution + end-to-end redaction** ‚Äî the fault-inject resource's
  `secret_ref` points at a **unique, high-entropy sentinel** (high-entropy is load-bearing:
  exact-value masking over a short/common value over-redacts and lets the "raw absent"
  assertion pass spuriously). The mock's `connect` (and/or `provision`) **resolves it through
  the runtime's `SecretBackend`** (which registers the value before returning it, ADR-0027)
  under a **per-op-unique scope identity**, then **emits the resolved value into a captured
  console transcript**. The transcript flows the **normal persistence path** through the
  `Redactor` built from the same registry, so the validation asserts the persisted artifact
  and any response snippet **both** lack the raw sentinel **and** carry the **redaction
  placeholder** ‚Äî the exact-value masking half of the contract that local-libvirt (which
  resolves no secrets) never exercised. The scope is **released only after** redact-and-persist
  (release-before-persist would evict the value and write the artifact unredacted, the exact
  leak this catches); the test asserts the value is gone from the snapshot after release
  (ADR-0073).
- **The four validation targets** ‚Äî reconciler/teardown drift repair, the secret loop above,
  admission-control concurrency races, and cancel/compensation semantics, each a thin test-PR
  over the settled mock seam (see ¬ßValidation surface).

## Non-goals (scoped out)

- **No real remote provider** ‚Äî that is M2; M1.5's only second provider is the in-process
  mock. The registry it builds is what M2 reuses (ADR-0071).
- **No capability dispatch** ‚Äî the registry is **static-kind** selection (a `dict` populated
  at composition time), not the `(plane, operation, resource_kind)` capability matching
  ADR-0063 deferred and ADR-0066 removed. M1.5 does not revive it (ADR-0071).
- **No new agent-facing MCP tools** ‚Äî faults are config on the fault-inject resource, not a
  tool; the resource is driven through the existing tool surface. Validation is operator/CI
  test harness, not new primitives (ADR-0072).
- **No new `ErrorCategory`** ‚Äî every injected fault maps to an **existing** taxonomy value;
  M1.5 adds none (contrast M1.4's `queue_timeout`). Picking the most specific existing value
  is the rule (ADR-0072).
- **No probabilistic faults without a configured seed** ‚Äî determinism is mandatory; a
  reachable `os.urandom`/wall-clock seed is a tested-against defect, not a fallback (ADR-0072).
- **No IOMMU / `<hostdev>` / real boot** ‚Äî the mock boots nothing real; M1.4's deferred
  passthrough wiring stays deferred. Synthetic outputs are plausible, not real infrastructure.

## Postgres schema (M1.5 delta)

- **One migration, `0018_resources_kind_fault_inject.sql`** ‚Äî widen the `resources_kind_check`
  CHECK from `IN ('local-libvirt')` to `IN ('local-libvirt', 'fault-inject')`, mirroring how
  `0003_reprovision_job_kind.sql` widened `jobs_kind_check`. Forward-only (ADR-0015). **No
  other DDL.**
- **No new columns or tables.** The fault-inject resource's `seed`, the per-plane `fault_rate`
  map, the per-plane `max_latency_s` bound, and `secret_ref` are **keys in the existing
  `resources.capabilities` jsonb**, set by discovery exactly like the existing `vcpus` /
  `memory_mb` / `concurrent_allocation_cap` keys. A fault-inject resource is a normal
  `resources` row whose `kind` is `fault-inject` and whose capabilities carry the fault-engine
  keys.

## Provider model (M1.5 delta)

`providers/composition.py` stays the **only** production provider-assembly point, but it now
assembles a **map**:

```
ResourceKind ‚îÄ‚îÄ‚ñ∂ ProviderRuntime
  local-libvirt ‚îÄ‚ñ∂ build_local_runtime()        # byte-identical to today
  fault-inject  ‚îÄ‚ñ∂ build_faultinject_runtime()
```

Worker handlers and **post-System** MCP tools that today close over *the* runtime instead
receive a `kind`-keyed **resolver**; each resolves from the System's Resource
(`job ‚Üí system ‚Üí allocation ‚Üí resource.kind`, non-null by then). **Resolution is scoped to
post-System ops only**: the pre-grant allocation plane (`allocations.request` / `.release`) is
core admission logic and a `requested` queued allocation has a **null `resource_id`**
(ADR-0069), so it resolves no runtime; discovery keys on the map entry's own kind, not a
Resource that does not yet exist. An implementer must not hang the resolver on a boundary that
can run before a System's Resource is fixed.

**The `fault-inject` entry is opt-in, absent from the default production composition.** The
map is assembled per deployment: production registers only `local-libvirt`; the `fault-inject`
entry **and its discovery registrar** are added only under an explicit config/env opt-in (the
CI/operator fault-injection stack). A default production deployment has **no** bookable
fault-inject Resource ‚Äî a tenant cannot allocate one or receive injected failures in prod
(ADR-0071). Discovery then fans out only over the **registered** map: local-libvirt registers
the local host; fault-inject registers a synthetic resource row **only in a fault-injection
deployment**. The provider port **interfaces are unchanged** ‚Äî the mock satisfies the same
`ProviderRuntime` dataclass ‚Äî so this is "no provider-seam change," consistent with the M2
falsifiability test.

## Validation surface (the four targets)

Each is a thin test-PR over the settled mock; none adds new platform behavior ‚Äî they
**exercise** existing behavior the mock can now reliably trigger.

- **Reconciler/teardown drift repair** ‚Äî each existing reconciler pass is driven by a fault
  and the exact repair asserted: **orphaned System** (provision succeeds, its allocation is
  then released/expired ‚Üí System torn down), **abandoned job** (worker-death simulated by
  lapsing the job heartbeat ‚Üí job marked abandoned + compensation runs), **dead DebugSession**
  (transport-drop fault on a `live` session ‚Üí reconciler moves it `detached`), **leaked
  provider infra** (mock `list_owned` reports a domain with no owning System row ‚Üí reaped),
  **lease-expiry-mid-job** (short lease + slow provision ‚Üí owning Run `failed(lease_expired)`,
  distinct from `canceled`).
- **Secret register‚Üíredact end-to-end** ‚Äî the ADR-0073 loop: resolve a high-entropy
  `secret_ref` under a per-op-unique scope, emit it into a captured transcript, assert the
  persisted artifact and response snippet are exact-value masked, and the scope is released
  **only after** redact-and-persist (not at a generic op end).
- **Admission-control concurrency races** ‚Äî concurrent `allocations.request` against the
  fault-inject resource contend the **per-project budget lock** and the **per-resource
  capacity lock** (ADR-0040); the mock keeps allocations active (injected latency / slow
  release) so contention is real. Assert **exactly one grants** at the cap boundary, **no
  budget overspend** (the check-then-debit is atomic, top-level design ¬ßConcurrency), and **no
  capacity double-book**.
- **Cancel/compensation semantics** ‚Äî `jobs.cancel` lands **mid-op** (the mock pauses via
  injected latency) on a half-done `provision`/`install`; each op's **declared** cancel policy
  (clean-rollback / best-effort / orphan-flagged) is exercised and asserted, proving "cancel
  is never undefined" (top-level design ¬ßReconciliation & teardown) against a provider that
  can be paused on demand.

## MCP tool surface (M1.5 delta)

**None.** M1.5 adds **no** agent-facing tools. The fault-inject resource is discovered and
registered service-side (like local-libvirt) and driven through the **existing** surface:
`resources.list` / `.describe` show it; `allocations.request` / `.release` claim it;
`systems.provision` / `runs.*` / `debug.*` / `control.*` exercise it. Faults are
`capabilities` config on the resource, not a tool call ‚Äî consistent with the seeded-chaos
control model (ADR-0072).

## Auth / RBAC delta

**None.** The fault-inject resource registers under the **service identity** at discovery, the
same path local-libvirt uses; no new role, claim, or gate. A caller allocates and drives a
fault-inject resource through the **existing** per-project `operator`/`viewer` checks ‚Äî it is
a Resource like any other. The admission-race tests run under ordinary project tokens (the
race is in the locks, not the authz).

## Error taxonomy (M1.5 delta)

**None.** Every injected fault maps to an **existing** `ErrorCategory`
(`PROVISIONING_FAILURE`, `INSTALL_FAILURE`, `BOOT_TIMEOUT`, `READINESS_FAILURE`,
`TRANSPORT_FAILURE`, `CONTROL_FAILURE`, `INFRASTRUCTURE_FAILURE`, `lease_expired`). The rule
is unchanged: pick the most specific existing value, invent no strings. M1.5 deliberately adds
no category ‚Äî the mock's job is to **trigger** the existing taxonomy, not extend it.

### Sequencing & shared seams (no separate plan)

M1.5 follows the M1.3/M1.4 model: detailed design lives here and in the three ADRs; each issue
is planned and implemented end-to-end by `work-issue`. There is **no
`docs/plans/m1.5-implementation.md`**. The cross-issue concerns no single issue owns are pinned
here:

- **One migration, claimed up front.** Issue **2** owns `0018_resources_kind_fault_inject.sql`
  (the only DDL this milestone) ‚Äî the CHECK widen lands with the fault-inject runtime it admits,
  so the issue-1 CHECK‚Üîregistry parity test never sees a CHECK-allowed kind without a runtime.
  No parallel-claim collision ‚Äî M1.5 has a single schema-touching issue, so no renumber-on-rebase
  is expected.
- **Land the spine before its consumers.** Issues **1 ‚Üí 2 ‚Üí 3** are strictly serial (the
  registry, then the mock that registers into it, then the engine that drives the mock).
  Issues **5/6/7** all consume the fault engine (issue 3) and **4** consumes the mock (issue
  2); none may start before its dependency merges. The merge wave is: **1**, then **2**, then
  **3 and 4** (4 needs only 2, so it parallels 3), then the fan-out **5/6/7**.
- **Expected rebase zones** (recurring in M1.3/M1.4 parallel work): `app.py` /
  `composition.py` provider wiring (issue 1 reshapes it; later issues touch registration),
  `tests/db/test_migrate.py` (issue 1's migration), and the generated `docs/guide/reference/*`
  if any tool docstring shifts (regenerated by `docs-check` ‚Äî rebase, don't hand-edit). Treat
  as merge-time rebases, not design conflicts. Note: M1.5 adds **no** tools, so the tool-doc
  churn that dominated M1.3/M1.4 rebases is largely absent.
- **Each issue runs in its own worktree** (parallel `work-issue` requires it, per the global
  standard) ‚Äî never share the working copy.

## Carried invariants

1. **The provider seam is unchanged** (ADR-0063) ‚Äî the mock satisfies the same
   `ProviderRuntime` ports; M1.5 changes provider *selection* (a composition-module registry),
   not the port *interfaces*, so the M2 "adding a provider touches zero `core/*` and
   MCP-tool-surface lines" hypothesis still holds and is now testable.
2. **Faults are reproducible** (ADR-0072) ‚Äî a fault decision is a pure function of `(seed,
   system_id, plane, attempt, facet)` over a **process-independent** hash, with `attempt`
   read from durable state; the same seed yields the same faults across runs regardless of
   worker concurrency, so every validation test is deterministic. A nondeterministic seed
   *or* a process-salted hash *or* a process-local `attempt` counter is a defect, not a
   fallback.
3. **A resolved secret is registered before it can be emitted, and released only after
   redact-and-persist** (ADR-0027, ADR-0073) ‚Äî `SecretBackend.resolve()` registers before
   returning, the mock emits only after resolving, the `Redactor` masks by exact value on
   persist, and the per-op-unique scope releases **after** every artifact/snippet that could
   contain the value is redacted and persisted ‚Äî so no resolved value reaches the object store
   or a response snippet unmasked, and none lingers as a global redaction needle past its op.
4. **Admission stays fail-closed and race-free** (ADR-0040) ‚Äî concurrent requests against the
   fault-inject resource serialize on the per-project budget lock and per-resource capacity
   lock; the mock makes the contention real but changes none of the admission logic it
   stresses.
5. **fault-inject is opt-in; default production registers only local-libvirt** (ADR-0071) ‚Äî
   the `fault-inject` map entry and its discovery registrar are composed only under an explicit
   opt-in, so a default production deployment has no bookable fault-inject Resource and a
   tenant cannot allocate one or receive injected failures in prod.

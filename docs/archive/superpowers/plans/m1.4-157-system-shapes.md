# Plan — #157 system_shapes table + migration + model + seed + resolver

Foundational M1.4 issue (no deps). Owns migration `0013`. ADR-0067 + the M1.4 spec
"Postgres schema" section are authoritative; this plan only sequences the build. No MCP
tool surface (that is #160/#161).

## Deliverables

1. **Migration `0013_system_shapes.sql`** (forward-only, additive to 0012):
   - Table `system_shapes`:
     - `name text PRIMARY KEY`
     - `vcpus int NOT NULL`
     - `memory_mb int NOT NULL` with `CHECK (memory_mb % 1024 = 0)` (whole-GB multiple,
       so `memory_mb → memory_gb` maps exactly, ADR-0067) — also `CHECK (memory_mb > 0)`.
     - `disk_gb int NOT NULL` (`CHECK (disk_gb > 0)`), `vcpus` (`CHECK (vcpus > 0)`)
     - `pcie_match text NULL` (stored opaquely; matcher lands in #158)
     - `updated_at timestamptz NOT NULL DEFAULT now()`
   - `set_updated_at` trigger (mirrors the accounting tables in 0002).
   - Data-migration seed `small`/`medium`/`large`/`max` (INSERT), bisectable like the
     cost-class seed. NO seed-time host-cap check (over-large is inert, fails closed later
     via `validate_against_resource`).

2. **`SystemShape` domain model** (`domain/models.py`) — `name`, `vcpus`, `memory_mb`,
   `disk_gb`, `pcie_match: str | None`, `updated_at`. Whole-GB validation on `memory_mb`
   at the model layer too (matches migration CHECK).

3. **Repository read** — a `KeyedRepository`/`Repository` instance over `system_shapes`
   keyed on `name`, exposing `get`. (Upsert/list belong to #160; this issue adds the read.)

4. **Resolver** (`domain/shapes.py`, mirroring `domain/cost.py::resolve_coeff`):
   - `async def resolve_shape(conn, name) -> ShapeSizing` returning the sizing tuple
     `{vcpus, memory_mb, disk_gb, pcie_match?}`.
   - Unknown name → fail-closed `CategorizedError(CONFIGURATION_ERROR)`.
   - A shape fixes size only; `cost_class` stays host-resolved (not in the tuple).

## Seed sizing

Dev host caps in discovery tests are ~8 vcpus / 16384 MB. Seeds (whole-GB memory):

| name   | vcpus | memory_mb | disk_gb |
|--------|-------|-----------|---------|
| small  | 1     | 1024      | 10      |
| medium | 2     | 4096      | 20      |
| large  | 4     | 8192      | 40      |
| max    | 8     | 16384     | 80      |

`pcie_match` NULL for all seeds. These should be ≤ the M1 dev host but that is not
enforced at seed time (ADR-0067).

# Plan — #158 PCIe descriptor inventory + discovery + match-spec matcher (M1.4 issue 4)

Authoritative design: ADR-0068 + `docs/specs/m1.4-system-catalog-scheduling.md`. This plan
covers only the implementation; the design is settled and not re-litigated here.

## Goal

Land the foundational, reusable PCIe **match-spec matcher** plus the host
`capabilities.pcie_devices` **static descriptor** inventory and its discovery population.
No request/availability/claim wiring — that is #162/#163/#159, which import this matcher.

## Deliverables

1. **Domain module `src/kdive/domain/pcie.py`** — the reusable, provider-agnostic core:
   - `PCIE_DEVICES_KEY = "pcie_devices"` capability key (next to
     `CONCURRENT_ALLOCATION_CAP_KEY`, but PCIe-specific so it lives here).
   - `PCIeDescriptor(TypedDict)` — `{bdf, vendor_id, device_id, class_code, label}`, all
     `str`. No `free` flag (ADR-0068: occupancy is derived from claims, never stored).
   - `PCIeClaim(TypedDict)` — the active-claim snapshot the matcher subtracts:
     `{vendor_id, device_id, bdf}`. The matcher consumes claims by **bdf** (a claim holds a
     resolved host-local device).
   - `parse_match_spec(spec: str) -> MatchSpec` — parse + validate grammar; raise
     `CategorizedError(CONFIGURATION_ERROR)` on malformed input (never an uncaught
     exception). Grammar (lowercase, exact):
     - `vendor:device` = `^[0-9a-f]{4}:[0-9a-f]{4}$` → exact (vendor_id, device_id).
     - `class=NN` = `^class=[0-9a-f]{2}$` → class high-byte (first 2 hex of class_code).
     - `class=NNNN` = `^class=[0-9a-f]{4}$` → class+subclass exact (first 4 hex).
     - Uppercase hex is **rejected** (spec says lowercase; fail-closed, no silent
       normalization — keeps the wire form canonical so two equal specs always string-match).
   - `descriptor_matches(spec, descriptor) -> bool` — predicate for one descriptor.
   - `resolve_spec(spec, descriptors, claims) -> SpecResolution` — resolve ONE spec against
     descriptors minus claimed bdfs. Returns a structured result distinguishing:
     - `MATCHED` (>=1 free descriptor) with the free candidates,
     - `CONFIG` (no descriptor anywhere matches the spec → card not in fleet),
     - `CAPACITY` (descriptors match but all are claimed → busy).
     Distinction is in the RETURN VALUE, not raised.
   - `resolve_multiset(specs, descriptors, claims) -> MultisetResolution` — resolve a list
     of specs to **distinct** free devices (two `8086:1572` specs → two different bdfs).
     Greedy assignment; each assignment removes the chosen bdf from the free pool. Reports
     per-spec outcome incl. a `CAPACITY` when a spec runs out of distinct free candidates
     even though the catalog has the model (e.g. one card, two specs). `parse_match_spec`
     failures still raise `CONFIGURATION_ERROR` (grammar is pre-resolution).

2. **Discovery population** — `LocalLibvirtDiscovery.list_resources()` adds the
   `pcie_devices` descriptor list, populated from libvirt **nodedev** PCI inventory
   (`listAllDevices(VIR_CONNECT_LIST_NODE_DEVICES_CAP_PCI_DEV)` → per-device `XMLDesc()`),
   parsed with `defusedxml` (trust boundary, mirroring `_parse_arch`). The nodedev PCI XML
   carries `<class>0xCCSSPP</class>`, `<vendor id='0xVVVV'>Label</vendor>`,
   `<product id='0xDDDD'>Label</product>`, and `<domain>/<bus>/<slot>/<function>`. Writes
   only the static descriptor; never an occupancy flag (idempotent vs. live claims).
   Field normalization (all lowercase, `0x` stripped):
   - **bdf — `<bus>/<slot>/<function>` are DECIMAL integers** (verified: `<slot>31</slot>`
     == BDF `1f`). Compose hex: `f"{dom:04x}:{bus:02x}:{slot:02x}.{fn:x}"` → `0000:00:1f.0`.
     Stringifying the decimals would yield a wrong, non-`lspci` bdf — explicit hex format.
   - **class_code** — store the full normalized **6-hex** form (`060100`); the matcher
     slices the first 2 (high byte) or first 4 (class+subclass) hex. Width pinned so
     `descriptor_matches` is deterministic.
   - **vendor_id/device_id** — 4-hex from the `id='0x...'` attributes.
   - **label** — `<product>` element **text**; `<product id='0x7a8a'/>` is self-closing
     with no text (verified), so fall back to `vendor:device` when text is empty/absent.
   - **Malformed single device — skip, don't blank the scan.** A per-device parse that
     fails (missing required field, malformed XML) is logged and the device skipped so one
     bad device never empties the inventory; an *attack* document (entity expansion) still
     raises via defusedxml (loud, mirroring `_parse_arch`). Both behaviors tested.
   - `_LibvirtConn` Protocol gains `listAllDevices`; a `_LibvirtNodeDevice` Protocol gets
     `name()`/`XMLDesc()`. `FakeLibvirtConn`/a `FakeNodeDevice` in tests inject canned XML.
   - Capabilities still uses the existing `dict[str, Any]` shape; `pcie_devices` is a list
     of `PCIeDescriptor` dicts.

3. **Tests** (TDD, failing first):
   - `tests/domain/test_pcie.py` — matcher unit tests driving functions directly with
     injected descriptor lists + claim sets: exact match, class 2-hex high-byte, class
     4-hex exact, multiset → distinct devices, spec absent everywhere → CONFIG, spec whose
     only matches are claimed → CAPACITY, multiset out-of-distinct-cards → CAPACITY,
     malformed (`zzzz:0000`, `class=xyz`, `class=`, `8086:1572:`, empty, uppercase
     `8086:1572`→ rejected) → `configuration_error`, no exception leaks.
   - `tests/providers/local_libvirt/test_discovery.py` — extend: discovery yields
     descriptors from injected nodedev XML; class/vendor/device/bdf/label parsed correctly
     incl. the **decimal→hex bdf** case (slot 31 → `1f`); empty `<product>` text falls back
     to `vendor:device`; a single malformed device is skipped (inventory of the rest
     survives); no `free` key present.

## Decisions / deviations (flagged)

- **Label source = nodedev `<product>` text, not a separate `lspci` subprocess.** The
  issue says "nodedev + lspci for the label"; nodedev's product element already carries the
  human label, so shelling out to `lspci` adds an untrusted-subprocess surface and an
  external-binary dependency for no extra data on this provider. Using nodedev keeps the
  seam injectable (no subprocess to mock) and is strictly less attack surface. Label text
  is still untrusted (libvirtd output) and is stored as an opaque string; consumers must
  treat it as display-only. If a later host genuinely lacks product text, label falls back
  to the bare `vendor:device` id pair.
- **`bdf` canonical form** = `DDDD:BB:SS.F` (domain:bus:slot.function, lowercased hex),
  matching `lspci -D` / nodedev path form, so a later claim snapshot is comparable.
- Matcher lives in `domain/` (provider-agnostic, reused by 3 MCP surfaces), not in the
  libvirt provider.

## Out of scope (explicit)

No `pcie_claim` persistence/DDL, no `allocations.request` wiring, no availability/systems
filter, no `<hostdev>`/vfio. No migration file (capabilities is existing jsonb).

## Guardrails

`just lint` / `just type` (whole-tree) / `just test` green before each commit; `just ci`
before finalizing. Conventional commits, one logical change each.

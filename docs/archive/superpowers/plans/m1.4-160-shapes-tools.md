# Plan ‚Äî #160: shapes.list + operator shapes.set/shapes.delete (M1.4)

ADR-0067, spec `docs/specs/m1.4-system-catalog-scheduling.md`. Builds on #157 (table/model/resolver) and #158 (`parse_match_spec`).

## Tool surface
- `shapes.list()` ‚Äî `viewer`. The spec says "viewer on any project"; the `system_shapes`
  catalog has **no `project` dimension** (shared fleet config), so that gate collapses to "any
  authenticated context" ‚Äî exactly the `resources.list` precedent (ADR-0023: shared infra
  reads need only an authenticated context, no RBAC scoping). Returns a `collection` envelope,
  one item per shape, sorted by name.
- `shapes.set(name, vcpus, memory_mb, disk_gb, pcie_match?)` ‚Äî `platform_operator`, upsert.
  Mirrors `ops.set_host_capacity`: `require_platform_role(PLATFORM_OPERATOR)`, denial audited
  iff the caller holds ‚â•1 platform role. Validates: non-empty `name`; `vcpus/memory_mb/disk_gb`
  positive; `memory_mb` whole-GB multiple of 1024; `pcie_match` (when present) parses via
  `kdive.domain.pcie.parse_match_spec` (raises `CONFIGURATION_ERROR` on bad grammar).
  Success ‚Üí audit row to `platform_audit_log`.
- `shapes.delete(name)` ‚Äî `platform_operator`. Deletes the catalog row. `shape` on
  allocations/systems is a label (no FK), so delete never FK-blocks or orphans a live row;
  existing rows keep their stamped sizing. Unknown name ‚Üí `configuration_error`.
  **Audit symmetry (matches `ops.set_host_capacity`):** a *successful* delete audits;
  unknown-name failure does **not** audit (nothing changed); a role denial audits iff the
  caller holds ‚â•1 platform role (`audit_platform_denial`).

## Repository
- Promote `SYSTEM_SHAPES` from `Repository` to `KeyedRepository` (`key_column="name"`,
  upsert every non-key column; `updated_at` server-generated/excluded ‚Äî same as QUOTAS).
- Add `list_all(conn)` and `delete(conn, key) -> bool` methods. `list_all` belongs on the
  base `Repository` (ordered by key_column) so it's reusable; `delete` on `Repository` too
  (keyed by `key_column`, returns whether a row was removed). The resolver's `get` path is
  unchanged. Constrain `name` non-empty on the write path (`SystemShape` model has no
  min-length on `name`; the tool validates before upsert).

## Module
- `src/kdive/mcp/tools/catalog/shapes.py` (catalog plane; list is a catalog read, set/delete
  are operator knobs over the catalog ‚Äî keep them co-located, mirroring tuning's single
  module for two ops knobs + register()).
- Register in `src/kdive/mcp/app.py` via `_plain(shapes.register)`.

## Tests ‚Äî `tests/mcp/catalog/test_shapes_tools.py`
Drive handlers directly with injected pool + RequestContext (the repo unit contract).
- list returns the 4 seeds, sorted, viewer context.
- set inserts a new shape; set updates an existing (upsert); both audited.
- set rejects non-multiple memory_mb ‚Üí configuration_error; not applied; not audited.
- set rejects malformed pcie_match grammar ‚Üí configuration_error; not applied.
- set stores a valid pcie_match.
- set rejects blank name / non-positive dims.
- set denied for project-only token (unaudited) and platform_auditor (denied + audited).
- delete removes the row + audited; delete unknown ‚Üí configuration_error AND writes no audit row.
- delete succeeds with live allocations/systems present and leaves those rows byte-for-byte
  unchanged. NB: there is no `shape` column on allocations/systems today (it lands in a later
  M1.4 issue), so there is no FK by construction ‚Äî the test asserts non-interference, not a
  live label reference, and the comment says so (no vacuous "FK-block" claim).

# M2.2 — Admin CLI (`kdivectl`) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship `kdivectl`, an authenticated MCP-client CLI that lets platform operators inspect and administer kdive state over the same OIDC/RBAC/audit boundary the agents use — replacing ad-hoc `psql`/`mc` poking.

**Architecture:** `kdivectl` is a FastMCP *client* (no new server transport). A generic client core attaches an OIDC bearer token and calls existing MCP tools; curated operator verbs render results as tables/JSON; a read-only fail-closed `tool call` passthrough reaches the rest of the surface. The only server-side change is audit attribution: the middleware records the caller's OIDC `client_id` and resolves an `actor` (`operator-cli`/`agent`/`unknown`). Mutating verbs route through the M1.3 break-glass tools. Two net-new read tools (`secrets.list` platform-gated, `fixtures.list` project-scoped) fill catalog gaps.

**Tech Stack:** Python 3.13, `uv`, `fastmcp-slim[client]==3.4.0` (already a dependency), `psycopg`/Postgres, `pytest`, `ty`, `ruff`. Spec: [`../specs/2026-06-10-m22-admin-cli-design.md`](../specs/2026-06-10-m22-admin-cli-design.md). ADR: [`../../adr/0089-operator-cli-mcp-client.md`](../../adr/0089-operator-cli-mcp-client.md).

## Grounding facts (verified against the tree)

- **Tool annotations already carry read/write intent.** `src/kdive/mcp/tools/_docmeta.py` exposes `read_only()` → `ToolAnnotations(readOnlyHint=True)`, `mutating()`, `destructive()`. Read tools already register with `annotations=_docmeta.read_only()`. The passthrough's "read-only" gate is therefore **`readOnlyHint is True`** read from existing metadata — *not* a new `mutating: bool` field. Fail-closed = treat missing/`False`/`None` as not-read-only.
- **Audit insert.** `src/kdive/security/audit.py::record_platform` inserts into `platform_audit_log (principal, agent_session, platform_role, tool, scope, args_digest)`. Issue 0 adds an `actor` column + value. Rows are built through two shared chokepoints — `mcp/tools/ops/_auth.py::audit_platform_denial` (denials) and `mcp/tools/ops/_reads.py` (cross-project read successes) — plus a few inline per-tool success sites; `actor` threads through all.
- **Platform-role implication (verified).** `security/authz/rbac.py::_PLATFORM_IMPLIES = {PLATFORM_ADMIN: {PLATFORM_AUDITOR}}`. `require_platform_role(ctx, role)` is satisfied only if the role is held or implied — so `platform_admin` does **not** imply `platform_operator`. Operator tools (and `secrets.list`) gate on `PLATFORM_OPERATOR`; operator tokens carry it explicitly. `require_platform_role` raises `AuthorizationError`, audited at the call site via the `except AuthorizationError: audit_platform_denial(...)` pattern (e.g. `ops/tuning.py:60-62`), **not** by middleware.
- **Migrations** live in `src/kdive/db/schema/NNNN_*.sql`; latest is `0020`. Issue 0 adds `0021_platform_audit_actor.sql`.
- **Platform gating.** `src/kdive/security/authz/rbac.py::require_platform_role(ctx, PlatformRole.PLATFORM_*)`; `PlatformRole` is a `StrEnum` (`PLATFORM_ADMIN`, `PLATFORM_OPERATOR`, `PLATFORM_AUDITOR`).
- **Secret presence.** `src/kdive/security/secrets/secret_registry.py::SecretRegistry.snapshot() -> frozenset[str]` is the presence surface (keys only, never values).
- **Token minting (tests).** `tests/integration/live_stack/harness.py::mint_token` drives the mock-oauth2-server auth-code flow with a literal `claims` JSON; `tests/mcp/conftest.py::mint` + `tests/mcp/roles.py` mint signed JWTs for unit tests. Both currently omit `client_id`/`azp` in claims — issue 0 extends them.
- **Config registry.** `src/kdive/config/registry.py::Setting[T]` (`name`, `parse`, `default`, `processes`, `group`, `help`, `suggest`); read via `kdive.config.get(SETTING)`.

```python
# src/kdive/security/authz/actor.py
"""Closed actor-attribution map for audit rows (ADR-0089 decision 5).

Resolves the *caller class* from the OIDC client id and agent-session presence.
Fail-safe: anything that is neither the configured CLI client nor a recognised
agent-with-session is ``unknown`` — never silently attributed to ``agent``.
"""

from __future__ import annotations

OPERATOR_CLI = "operator-cli"
AGENT = "agent"
UNKNOWN = "unknown"


def resolve_actor(client_id: str | None, *, agent_session: str | None, cli_client_id: str) -> str:
    """Return the audit ``actor`` for a verified token's ``client_id``/``agent_session``."""
    if client_id is not None and client_id == cli_client_id:
        return OPERATOR_CLI
    if agent_session:
        return AGENT
    return UNKNOWN
```

```python
# src/kdive/cli/transport.py
"""kdivectl's MCP-client session: resolve URL, attach bearer token, call a tool.

Imports NO kdive.services and reads NO DB/object-store settings — the operator host
holds only the bearer token (ADR-0089 decision 5). Enforced by test_no_service_import.
"""

from __future__ import annotations

from dataclasses import dataclass

from fastmcp import Client
from fastmcp.client.auth import BearerAuth

import kdive.config as config
from kdive.config.cli_settings import SERVER_URL, TOKEN
from kdive.cli.login import read_cached_token


@dataclass(frozen=True)
class Session:
    url: str
    token: str

    @classmethod
    def from_env(cls) -> "Session":
        token = config.get(TOKEN) or read_cached_token()
        if not token:
            raise SystemExit("no token: run `kdivectl login` or set KDIVE_TOKEN")
        return cls(url=config.get(SERVER_URL), token=token)

    async def call(self, tool: str, arguments: dict) -> object:
        async with Client(self.url, auth=BearerAuth(self.token)) as client:
            return await client.call_tool(tool, arguments)
```

(`read_cached_token` is defined in Task 2; until then, stub it to `return None` and replace.)

```python
# src/kdive/cli/passthrough.py
"""Read-only, fail-closed gate for `kdivectl tool call` (ADR-0089 decision 2a).

Allows only tools whose MCP `readOnlyHint` is True; anything mutating, unannotated,
or unknown is refused so the passthrough is never a route around break-glass.
"""

from __future__ import annotations


class NotReadOnlyError(RuntimeError):
    """Raised when a tool is not positively annotated read-only."""


def assert_read_only(name: str, tool: object) -> None:
    """Raise unless ``tool`` carries ``annotations.readOnlyHint is True``."""
    annotations = getattr(tool, "annotations", None)
    if getattr(annotations, "readOnlyHint", None) is not True:
        raise NotReadOnlyError(
            f"{name!r} is not read-only; mutations go through curated break-glass verbs"
        )
```

```python
"""`kdivectl` entry point: parse args, dispatch to login / tool call / curated verbs."""

from __future__ import annotations

import argparse
import asyncio

def build_parser() -> argparse.ArgumentParser:
    parser = argparse.ArgumentParser(prog="kdivectl")
    parser.add_argument("--json", action="store_true", help="machine-readable JSON output")
    sub = parser.add_subparsers(dest="command", required=True)
    login = sub.add_parser("login", help="acquire and cache a bearer token")
    login.add_argument("--platform-role", choices=["platform_admin", "platform_operator"], default=None)
    tool = sub.add_parser("tool", help="generic MCP passthrough (read-only)")
    tool_sub = tool.add_subparsers(dest="tool_command", required=True)
    call = tool_sub.add_parser("call")
    call.add_argument("name")
    call.add_argument("--json", dest="payload", default="{}")
    return parser

def main(argv: list[str] | None = None) -> int:
    args = build_parser().parse_args(argv)
    from kdive.cli import dispatch  # late import keeps `--help` cheap
    return asyncio.run(dispatch.run(args))

if __name__ == "__main__":
    raise SystemExit(main())
```

```python
# src/kdive/cli/login.py
"""`kdivectl login`: acquire a bearer token (mock-OIDC) and cache it 0600 (ADR-0089).

The token is written to a per-user file created 0600 (parent 0700) and is never logged.
Production operators bring their own token via KDIVE_TOKEN; this verb targets dev/CI and
the boundary test, parameterized on the platform-role axis.
"""

from __future__ import annotations

import os
from pathlib import Path


def _cache_path() -> Path:
    base = Path(os.environ.get("XDG_STATE_HOME", Path.home() / ".local" / "state"))
    return base / "kdive" / "token"


def write_cached_token(token: str) -> None:
    path = _cache_path()
    path.parent.mkdir(mode=0o700, parents=True, exist_ok=True)
    fd = os.open(path, os.O_WRONLY | os.O_CREAT | os.O_TRUNC, 0o600)
    with os.fdopen(fd, "w") as handle:
        handle.write(token)
    os.chmod(path, 0o600)  # tighten if a prior umask widened it


def read_cached_token() -> str | None:
    try:
        return _cache_path().read_text().strip() or None
    except FileNotFoundError:
        return None
```

```python
# src/kdive/cli/render.py
"""Operator-facing output: a plain aligned table by default, stable JSON with --json."""

from __future__ import annotations

import json
from collections.abc import Sequence


def render(rows: Sequence[dict], *, columns: Sequence[str], as_json: bool) -> None:
    """Render a list of rows as an aligned table, or stable JSON."""
    if as_json:
        print(json.dumps(list(rows), indent=2))
        return
    widths = {c: max(len(c), *(len(str(r.get(c, ""))) for r in rows)) if rows else len(c) for c in columns}
    print("  ".join(c.ljust(widths[c]) for c in columns))
    for row in rows:
        print("  ".join(str(row.get(c, "")).ljust(widths[c]) for c in columns))


def render_record(record: dict, *, as_json: bool) -> None:
    """Render a single record as aligned key/value lines, or stable JSON.

    The single-record verbs (`describe`/`get`/`show`) return one dict, not a row list.
    """
    if as_json:
        print(json.dumps(record, indent=2))
        return
    width = max((len(k) for k in record), default=0)
    for key, value in record.items():
        print(f"{key.ljust(width)}  {value}")
```

```python
# src/kdive/cli/commands/__init__.py
"""Registry mapping (group, subcommand) -> a CLI verb handler + its argparse shape."""

from __future__ import annotations

from dataclasses import dataclass, field
from collections.abc import Awaitable, Callable

from kdive.cli.commands import reads


@dataclass(frozen=True)
class Verb:
    group: str
    sub: str
    handler: Callable[[object], Awaitable[int]]
    positionals: tuple[str, ...] = ()      # e.g. ("resource_id",) for describe/get/show
    kind: str = "list"                      # "list" -> render(); "record" -> render_record()


REGISTRY: tuple[Verb, ...] = (
    Verb("resources", "list", reads.resources_list),
    Verb("resources", "describe", reads.resources_describe, ("resource_id",), kind="record"),
    Verb("allocations", "list", reads.allocations_list),
    Verb("allocations", "get", reads.allocations_get, ("allocation_id",), kind="record"),
    # ... systems/runs/jobs/ledger/inventory per the Step-5 table; mutating verbs (Task 5)
    #     append their own Verb entries to this same REGISTRY.
)


def add_subparsers(sub) -> None:
    """Add one subparser per registry group, with a sub-subparser per verb."""
    groups: dict[str, object] = {}
    for verb in REGISTRY:
        group_parser = groups.get(verb.group)
        if group_parser is None:
            group_parser = sub.add_parser(verb.group).add_subparsers(dest="subcommand", required=True)
            groups[verb.group] = group_parser
        verb_parser = group_parser.add_parser(verb.sub)
        for positional in verb.positionals:
            verb_parser.add_argument(positional)


async def run_verb(args) -> int:
    """Resolve (command, subcommand) against the registry and await its handler."""
    for verb in REGISTRY:
        if verb.group == args.command and verb.sub == getattr(args, "subcommand", None):
            return await verb.handler(args)
    raise SystemExit(f"unknown command: {args.command} {getattr(args, 'subcommand', '')}")
```

```python
"""`secrets.list` — secret *presence* (refs), platform-role gated (ADR-0089 decision 6).

Off the general agent surface: presence is a reconnaissance primitive. Returns ref keys
only — never values; the SecretRegistry holds values, this reads its key snapshot.
Audits the cross-platform read (success and overreach denial) like every ops read.
"""

from __future__ import annotations

from fastmcp import FastMCP
from psycopg_pool import AsyncConnectionPool

from kdive.domain.errors import ErrorCategory
from kdive.mcp.auth import current_context
from kdive.mcp.responses import ToolResponse
from kdive.mcp.tools import _docmeta
from kdive.mcp.tools.ops._auth import ALL_PROJECTS_SCOPE, audit_platform_denial
from kdive.security import audit
from kdive.security.authz.actor import resolve_actor
from kdive.security.authz.rbac import AuthorizationError, PlatformRole, require_platform_role
from kdive.security.secrets.secret_registry import SecretRegistry
import kdive.config as config
from kdive.config.cli_settings import CLI_CLIENT_ID

_TOOL = "secrets.list"
_OBJECT_ID = "secrets"  # object_id for the envelope (no single record; the ref set is the object)


def _denied() -> ToolResponse:
    # Mirrors ops/tuning.py::_denied: failure(object_id, ErrorCategory, ...) → status="error".
    return ToolResponse.failure(
        _OBJECT_ID, ErrorCategory.AUTHORIZATION_DENIED, suggested_next_actions=[_TOOL]
    )


async def list_secrets_tool(pool: AsyncConnectionPool, registry: SecretRegistry, ctx) -> ToolResponse:
    try:
        require_platform_role(ctx, PlatformRole.PLATFORM_OPERATOR)
    except AuthorizationError:
        await audit_platform_denial(pool, ctx, tool=_TOOL, scope=ALL_PROJECTS_SCOPE)
        return _denied()
    refs = sorted(registry.snapshot())
    async with pool.connection() as conn, conn.transaction():
        await audit.record_platform(
            conn, principal=ctx.principal, agent_session=ctx.agent_session,
            event=audit.PlatformAuditEvent(
                tool=_TOOL, scope=ALL_PROJECTS_SCOPE, args={},
                platform_role=",".join(sorted(r.value for r in ctx.platform_roles)),
                actor=resolve_actor(ctx.client_id, agent_session=ctx.agent_session,
                                    cli_client_id=config.get(CLI_CLIENT_ID)),
            ),
        )
    return ToolResponse.success(_OBJECT_ID, "ok", data={"secrets": refs})


def register(app: FastMCP, pool: AsyncConnectionPool, registry: SecretRegistry) -> None:
    @app.tool(name=_TOOL, annotations=_docmeta.read_only(), meta={"maturity": "implemented"})
    async def secrets_list() -> ToolResponse:
        """List secret-ref *presence* (keys only). Platform-role gated; operator-only."""
        return await list_secrets_tool(pool, registry, current_context())
```

```python
# src/kdive/cli/commands/mutations.py
"""Mutating operator verbs: route through break-glass tools; fail-closed on token expiry.

Destructive verbs are single-call and re-runnable (idempotent against
already-released/already-torn-down state), so a refused near-expired token costs nothing
to re-acquire and re-run (ADR-0089).
"""

from __future__ import annotations

import base64
import json


class TokenExpiringError(RuntimeError):
    """The bearer token is too close to expiry to start a destructive action."""


def _decode_exp(token: str) -> int | None:
    try:
        body = token.split(".")[1]
        body += "=" * (-len(body) % 4)
        return json.loads(base64.urlsafe_b64decode(body)).get("exp")
    except Exception:
        return None


def ensure_token_valid(token: str, *, now: int, margin_s: int = 30) -> None:
    exp = _decode_exp(token)
    if exp is None or exp - now <= margin_s:
        raise TokenExpiringError("token missing exp or expiring soon; run `kdivectl login` and retry")
```

```python
# tests/integration/test_kdivectl_boundary.py  (require_live_stack, like the other spine tests)
import pytest

# NOTE: ops.force_release is PLATFORM_ADMIN-gated (breakglass.py:117) — role choice is
# verb-specific. The under-privileged principal must HOLD a platform role that is
# insufficient (here platform_operator) — NOT an empty role set: audit_platform_denial
# deliberately does NOT record a no-platform-role denial (the routine non-grant case), so
# an empty-role token would be denied but NOT audited, failing the assert.
@pytest.mark.require_live_stack
@pytest.mark.asyncio
async def test_mutating_verb_denied_for_underprivileged_and_audited(live_stack, mint_token, audit_rows):
    admin = mint_token(platform_roles=["platform_admin"], client_id="kdivectl")
    operator = mint_token(platform_roles=["platform_operator"], client_id="kdivectl")  # insufficient for admin-gated force_release, but a platform role

    # admin succeeds, audited as operator-cli
    await run_kdivectl(["allocations", "force-release", live_stack.allocation_id], token=admin)
    assert any(r["actor"] == "operator-cli" and r["tool"] == "ops.force_release"
               for r in await audit_rows())

    # under-privileged (holds platform_operator, not admin): denied AND audited
    code = await run_kdivectl(["allocations", "force-release", live_stack.allocation_id], token=operator)
    assert code == 3  # authorization_denied exit code
    assert any(r["actor"] == "operator-cli" and r["tool"] == "ops.force_release"
               for r in await audit_rows())  # denial row, actor recorded
```

# Plan ‚Äî M1.5 (7): cancel/compensation semantics (#186)

## Goal

Prove **"cancel is never undefined"** against the fault-inject provider: `jobs.cancel`
lands **mid-op** on a half-done `provision`/`install` (the mock pauses via injected
latency), and each op's **declared** cancel policy (`clean_rollback` / `best_effort` /
`orphan_flagged`, from #182's `cancel_policy.py`) is exercised so the observable
post-cancel state in the mock infra inventory is distinct per policy.

This is a `type:test` issue. The design is settled: `docs/specs/m1.5-fault-injection-provider.md`
¬ßValidation surface and ADR-0072 (cancel policy). The seeded engine
(`engine.py`), the per-op policy table (`cancel_policy.py`: `CancelPolicy` /
`CANCEL_POLICY` / `cancel_policy_for`), and the mock inventory (`inventory.py`) are
**already on main** ‚Äî this issue **consumes** them. No new ADR; no new agent tool.

## Design decisions (within the settled spec)

The provider ports (`provision`, `install`, ‚Ä¶) are synchronous and run under
`asyncio.to_thread`. `jobs.cancel` is **cooperative** (`jobs.py::cancel_job` flips the
Job row to `CANCELED`; nothing polls it mid-op today). To land a cancel *mid-op* on a
*half-done* op we need three things, none of which exist yet for the mock:

1. **A pause the cancel can land inside.** The op must block at a checkpoint after it
   has done its first observable side effect (e.g. provision has recorded the domain in
   the inventory) but before it finishes. ADR-0072 names *injected latency* as the
   cancel lever; we model that latency as a blocking wait on a **cancel signal** with a
   bounded timeout, so the test does not sleep on a wall clock (deterministic, fast CI).

2. **A cooperative cancel observation.** At the checkpoint the op polls a
   `CancelSignal` (a thin wrapper over `threading.Event`, since the op runs in a worker
   thread). If set, the op stops mid-flight and runs its declared compensation; if the
   timeout elapses without a cancel, the op completes normally (happy path unchanged).

3. **Per-policy compensation against the inventory.** On an observed cancel the op
   applies `cancel_policy_for(plane)`. The three states are distinguished by **what the
   reconciler's leaked-domain pass (`reconciler/loop.py::_repair_leaked_domains`) sees**
   ‚Äî entry presence plus one flag that pass can act on ‚Äî *not* by markers no production
   code reads. `_repair_leaked_domains` reaps every domain `list_owned()` returns whose
   System row is gone, so "entry present" is itself the residue the sweep handles:
   - `CLEAN_ROLLBACK` ‚Üí fully unwind: the partial side effect is removed; the inventory
     has **no** entry for the System. Nothing leaks; the reconciler finds nothing.
   - `BEST_EFFORT` ‚Üí "unwind what it can; residue is tolerated and swept later"
     (`cancel_policy.py`). Model faithfully as **residue left behind**: the domain entry
     **remains** in `owned_domains`, **un-flagged**. The later leaked-domain sweep reaps
     it like any leaked domain ‚Äî exactly "swept later." Distinct from clean (entry
     absent).
   - `ORPHAN_FLAGGED` ‚Üí "may leave provider state the reconciler reaps; it flags rather
     than guarantees rollback" (`cancel_policy.py`). The entry **remains** and is
     **orphan-flagged** (`flag_orphan`). The flag is a real signal: a reaper/operator can
     distinguish an *intentionally-orphaned* domain (cancel left it deliberately) from
     best-effort *tolerated residue*. Distinct from best-effort by the flag, from clean by
     entry presence.

   So the three states are: **(clean)** entry absent; **(best-effort)** entry present,
   unflagged; **(orphan)** entry present, flagged ‚Äî three genuinely distinct shapes, each
   grounded in the declared `cancel_policy.py` docstring, with no unread scaffolding.

   `provision`'s declared policy is `ORPHAN_FLAGGED`; `install`'s is `BEST_EFFORT`. To
   exercise all three declared *values* (acceptance: "each declared policy has a test"),
   a `CLEAN_ROLLBACK` plane (`connect`/`control`) is also exercised via the same runner ‚Äî
   so every distinct policy value has a pause‚Üícancel‚Üíassert test, not only the two planes
   the issue names by example.

### Ownership split

- **New module `cancel.py`** in `providers/fault_inject/`: owns the `CancelSignal`
  (cooperative signal + bounded wait) and the `run_cancelable_op` runner that threads
  the signal through one op and applies `cancel_policy_for(plane)` on an observed
  cancel. It imports `cancel_policy.py` and `inventory.py`; it does **not** import the
  worker or the MCP layer (the seam is provider-local and unit-testable, per
  `AGENTS.md`'s "drive the unit directly with injected dependencies").
- **Inventory gains one orphan flag.** `inventory.py` currently records only
  `{domain: system_id}`. Add a single `flag_orphan(domain)` / `is_orphaned(domain)` pair
  (an `_orphaned: set[str]`). No best-effort marker is added ‚Äî best-effort is modeled by
  *leaving the existing entry*, which needs no new state. Keep `record`/`owned_domains`
  unchanged so the happy-path spine test (`test_fault_inject_spine.py`) and the reaper are
  untouched; `forget` additionally drops the orphan flag so a torn-down domain leaves no
  stale flag.

### Why not drive through the real worker + `jobs.cancel`?

The acceptance is about the **provider's compensation** being well-defined per op, not
about re-testing the queue's cancel transition (already covered by `jobs` tests). Per
`AGENTS.md` (ADR-0019: drive handlers/units directly; don't go through transport), the
unit under test is the provider op + its policy, driven with an injected cancel signal.
A separate, thin test asserts the cancel *binding*: `cancel_policy_for(plane)` is total
(every plane defined) so no op is undefined ‚Äî that totality is already pinned in
`test_cancel_policy.py`; we add the *behavioral* half here.

## TDD steps (each step: red ‚Üí green ‚Üí guardrails ‚Üí commit)

### Step 1 ‚Äî Inventory orphan flag
- **Test (red):** in `test_inventory.py`, assert: `flag_orphan` makes `is_orphaned`
  true for that domain only (another recorded domain stays unflagged) and leaves the
  domain in `owned_domains`; `forget` clears the orphan flag (no stale flag ‚Äî a later
  `record`+`is_orphaned` of the same name is false); `flag_orphan` is idempotent; an
  unknown name is a no-op (`is_orphaned` false, no `KeyError`).
- **Impl (green):** add `_orphaned: set[str]` to `FaultInjectInventory`; `flag_orphan` /
  `is_orphaned`; have `forget` discard the name from `_orphaned`.
- Guardrails: `just lint && just type && just test`.

### Step 2 ‚Äî `CancelSignal` + bounded cooperative wait
- **Test (red):** in `test_cancel_semantics.py`, `CancelSignal.wait_for_cancel(timeout)`
  returns `True` promptly when the signal is set (from another thread), `False` when the
  timeout elapses with no cancel; `cancel()` is idempotent; a zero/negative timeout does
  not block.
- **Impl (green):** `CancelSignal` wraps `threading.Event`; `wait_for_cancel` is
  `event.wait(timeout)`; `cancel()` sets it.
- Guardrails.

### Step 3 ‚Äî `run_cancelable_op` happy path (no cancel)
- **Test (red):** with a signal never set and a small `latency_s`, the runner runs the
  op's first side effect, waits, and **completes** ‚Äî returning a **completed** outcome
  (not observed-cancel) and leaving the normal inventory state (domain recorded,
  unflagged). Asserts the op's result is returned unchanged. This pins the happy-path
  outcome that Step 5's best-effort state shares an inventory shape with ‚Äî so the two are
  told apart by the outcome, never by inventory alone.
- **Impl (green):** runner records the first side effect via the injected `op`, waits
  `wait_for_cancel(latency_s)`; on `False` returns completed.
- Guardrails.

### Step 4 ‚Äî `ORPHAN_FLAGGED` (provision) cancel mid-op
- **Test (red):** seed a `provision`-plane run; the op records the domain, the test sets
  the signal during the wait window, the runner observes cancel and applies
  `cancel_policy_for(PROVISION)` (`ORPHAN_FLAGGED`): the domain **stays** in
  `owned_domains` and `is_orphaned(domain)` is **true**. Outcome marks observed-cancel +
  the applied policy.
- **Impl (green):** dispatch on `cancel_policy_for(plane)`; orphan branch leaves the
  entry and calls `flag_orphan(domain)`.
- Guardrails.

### Step 5 ‚Äî `BEST_EFFORT` (install) cancel mid-op
- **Test (red):** `install`-plane; on cancel the runner leaves the residue ‚Äî the domain
  **stays** in `owned_domains` but `is_orphaned(domain)` is **false** (the later
  leaked-domain sweep reaps it like any leaked domain, i.e. "swept later"). The inventory
  shape (present, unflagged) is identical to a *healthy uncancelled* domain, so the
  inventory assertion alone cannot prove a cancel landed ‚Äî the test **must** also assert
  the returned `CancelOutcome` marks **observed-cancel** with applied policy
  `BEST_EFFORT`. That ties the residue to a proven mid-op cancel, not to "nothing changed."
- **Impl (green):** best-effort branch: leave the entry, do not flag; return an
  observed-cancel outcome carrying `BEST_EFFORT`.
- Guardrails.

### Step 6 ‚Äî `CLEAN_ROLLBACK` cancel mid-op
- **Test (red):** a `clean_rollback` plane (`connect`); on cancel the runner fully

- **Impl (green):** clean branch: `forget(domain)`.

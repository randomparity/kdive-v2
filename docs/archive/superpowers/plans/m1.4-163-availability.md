# Plan ‚Äî #163 `resources.availability` (M1.4, ADR-0070)

## Goal
Add a `viewer` read `resources.availability(pcie?, shape?)` reporting, per host: free
capacity headroom, free PCIe descriptors, and the shapes that fit now; plus fleet/kind
queue depth. Schedulability-aware. No migration (read-only aggregate).

## Authority / placement
- New handler module `src/kdive/mcp/tools/catalog/availability.py` (keeps `resources.py`
  from growing further; registered alongside the other `resources.*` tools). The tool
  name stays `resources.availability`; registration wired in `src/kdive/mcp/app.py`.
- `viewer` = any authenticated context (the `resources.list` / `shapes.list` precedent:
  shared infra has no `project` column, so no RBAC scoping). Hosts are not project-scoped,
  so the per-host view leaks nothing; queue depth is reported only as fleet/kind counts.

## Headroom (must match admission)
- Reuse the SAME occupancy predicate admission uses. The constant `_OCCUPYING` in
  `src/kdive/services/allocation_admission.py` (#164) is module-private; promote it to a
  shared importable name (`OCCUPYING` / `OCCUPYING_VALUES`) and have admission import the
  promoted symbol so availability and admission can never disagree (structural, not a
  copy). Per host: `headroom = cap ‚àí count(state ‚àà GRANTED/ACTIVE/RELEASING)`. Queued
  `requested` rows are excluded.
- Occupancy is one fleet-wide `SELECT resource_id, count(*) ... WHERE state = ANY(occupying)
  GROUP BY resource_id` (not N calls to the per-host `_count_occupying`), joined to the
  resource rows in Python ‚Äî a read-only aggregate, no N+1.
- Cap resolution: a host whose `concurrent_allocation_cap` is missing/invalid is the case
  admission's `_resolve_cap` *raises* `configuration_error` on (un-grantable). Availability
  must NOT point the agent at such a host: flag it **non-schedulable** (`schedulable=false`,
  `headroom=0`) so it never "fits now" ‚Äî mirroring ADR-0070's "never points the agent at a
  host every request would refuse." Degrade that one row; never crash the aggregate.

## Free PCIe descriptors
- `pcie_claim.descriptors_for(resource)` minus `pcie_claim.active_claims(conn, id)` by BDF.
  Count free descriptors; surface vendor/device/class but **redact `label`** (host-derived,
  untrusted) ‚Äî return the bdf/vendor/device only, never the opaque label string.

## Fits-now
- A shape "fits now" iff a **schedulable** host (`status == available` AND not `cordoned`)
  has BOTH `headroom >= 1` AND, when the shape carries a `pcie_match`, a free device the
  matcher resolves for that single spec (`resolve_spec`, MATCHED). Iterate
  `SYSTEM_SHAPES.list_all`; report fitting shape names.

## Queue depth (fleet/kind)
- Fleet total: an unconditional `count(*) WHERE state = 'requested'` ‚Äî grouping-independent,
  so it always counts every queued row.
- By-kind breakdown: `GROUP BY requested_kind`. NOTE a **by-id** queued row carries
  `requested_kind = NULL` (only `requested_resource_id` is set; admission sets exactly one,
  `allocations.py:247-248`). So the by-kind sum need not equal the total; report the by-id /
  NULL-kind rows under an explicit `by_id` (a.k.a. unattributed) tally so the breakdown is
  exhaustive and the total is never understated. Queued rows have `resource_id` NULL, so
  none of this is host-attributable.

## Filters
- `pcie` (optional match spec): narrows the per-host view to hosts with ‚â•1 free matching
  device (reuse `resolve_spec`); malformed spec ‚Üí `configuration_error` (parse raises).
- `shape` (optional name): resolve via `resolve_shape` (fail-closed `configuration_error`);
  restrict the fits-now computation / per-host fit to that one shape.

## Response shape
- A collection envelope (object_id `"resources"`), one item per host carrying:
  `headroom`, `cap`, `in_use`, `free_pcie` count, `schedulable` flag, `status`, `cordoned`,
  and the shapes that fit on that host. Top-level `data` carries fleet queue depth
  (total + by-kind) and the global fits-now shape set. Documented as a point-in-time hint,
  never a reservation.

## Tests (TDD, handler driven directly with injected pool)
- headroom = cap ‚àí occupancy (GRANTED/ACTIVE/RELEASING reduce; `requested` does not)
- free PCIe excludes active claims
- fits-now only on a schedulable host with both headroom AND a free matching device
- cordoned / non-available host flagged non-schedulable + never fits
- fleet total queue depth counts ALL `requested` rows; a by-kind AND a by-id queued row
  are both counted in the total, and the by-id row lands in the `by_id` tally
- invalid/missing-cap host is flagged non-schedulable and never fits now
- pcie filter narrows; shape filter narrows ‚Äî each in one call
- malformed pcie spec ‚Üí configuration_error; unknown shape ‚Üí configuration_error
- label redaction: untrusted host label never returned

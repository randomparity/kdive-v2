# Plan: systems.list view (#159)

## Goal
Add `systems.list(allocation_id?, state?, shape?, pcie?)` — a `viewer` read scoped to the
caller's projects, filterable by allocation, System state, shape (named OR a full-custom
sentinel mapping to `shape IS NULL`), and a PCIe match-spec evaluated against each System's
allocation `pcie_claim`. Mirrors `allocations.list` pagination + cap. No migration (both
columns exist on main).

## Surface
- Handler `list_systems(pool, ctx, *, allocation_id, state, shape, pcie, limit)` in
  `src/kdive/mcp/tools/lifecycle/systems/provision.py` (alongside `get_system`,
  `_envelope_for_system`). Tool `systems.list` registered in the systems registrar; wired
  through `src/kdive/mcp/app.py` (already routes the systems registrar).
- No `project` param: scope to `ctx.projects` filtered to those holding **a role ranking ≥
  viewer**. `ctx.projects` (the `projects` claim) and `ctx.roles` (the `roles` claim) are
  independent — a project can be a member project with **no** role (`roles_from_claims`
  returns `{}` for an absent claim). The predicate MUST guard `held is not None` before the
  rank compare (mirror `require_role`: `held is None or _RANK[held] < _RANK[VIEWER]`):
  `[p for p in ctx.projects if (r := ctx.roles.get(p)) is not None and _RANK[r] >= _RANK[VIEWER]]`.
  A bare member (no role) is omitted, same no-leak shape as an ungranted project. `_RANK[VIEWER]
  == 0`, so a naive `>= 0` without the None guard would leak to bare members — the guard is load-bearing.

## Scoping / no-leak (ADR-0023)
- `WHERE project = ANY(viewer_projects)`. If `viewer_projects` is empty → empty collection.
- A System in an ungranted project never appears in the query result set (omitted, not
  hidden in a field). No per-row authz leak — the row is simply absent.

## Filters
- `allocation_id`: parse to UUID; a malformed id → `configuration_error`. SQL `allocation_id = %s`.
- `state`: validate against `SystemState`; unknown value → `configuration_error`. SQL `state = %s`.
- `shape`: a sentinel string (`"__custom__"`) → `shape IS NULL` (full-custom); otherwise
  `shape = %s` (named). No catalog lookup — `shape` is a recorded label (ADR-0067).
- `pcie`: parse via `kdive.domain.pcie.parse_match_spec` (malformed/empty →
  `configuration_error`, the most-specific existing category). The match runs **in SQL** (a
  join to `allocations`, see below) so the cap is applied to the already-PCIe-filtered set —
  not after. `pcie_claim` is the *resolved vendor:device* snapshot (ADR-0068); a claim entry
  carries `vendor_id`/`device_id`/`bdf` but **no `class_code`**. Therefore:
  - **`vendor:device` spec** → SQL `EXISTS` over the joined allocation's `pcie_claim` jsonb
    array: `EXISTS (SELECT 1 FROM jsonb_array_elements(a.pcie_claim) e
    WHERE e->>'vendor_id' = %s AND e->>'device_id' = %s)`.
  - **`class=` spec** → **rejected** with `configuration_error` (class-level matching can't be
    honored against the claim snapshot — claims carry no class_code; a silent false-empty would
    mislead an agent into provisioning a new System instead of reusing one). The parse still
    runs first so a malformed spec is still `configuration_error`; a *well-formed* `class=` spec
    is then rejected with the same category and a clear message. Detect via the parsed
    `MatchSpec.kind`.

## Pagination / cap
- Same as `allocations.list`: `DEFAULT_LIST_LIMIT = 50`, `MAX_LIST_LIMIT = 200`,
  `capped = max(1, min(limit, MAX))`, `ORDER BY s.created_at DESC, s.id LIMIT %s`. **All**
  filters (allocation/state/shape/pcie) are SQL predicates, so the cap is applied over the
  fully-filtered set — there is no post-cap Python filtering that could undercount matches.

## Envelope
- `ToolResponse.collection("systems", "ok", [_envelope_for_system(s) for s in rows], ...)`.

## Tests (TDD, driven against the handler with an injected pool; migrated_url DB)
1. lists the caller's Systems (multiple projects the caller views).
2. `allocation_id` filter selects only that allocation's Systems.
3. `state` filter selects only that state.
4. `shape=NAME` filter selects only that named shape.
5. shape sentinel → returns only full-custom (`shape IS NULL`) Systems.
6. `pcie` vendor:device spec matches a System whose allocation `pcie_claim` contains the
   device; excludes a System whose allocation lacks it.
7. malformed `pcie` spec → `configuration_error`; empty/whitespace `pcie` → `configuration_error`.
8. well-formed `class=` `pcie` spec → `configuration_error` (unsupported against claim snapshot).
9. malformed `allocation_id` / unknown `state` → `configuration_error`.
10. cross-project System in an **ungranted** project omitted (no-leak).
11. System in a **member project with no role** omitted (no-leak; distinct from #10) —
    the `held is not None` guard.
12. pagination/cap honored (limit clamps to MAX, min 1); a match beyond the cap is
    correctly counted because filtering is in SQL (no post-cap undercount).
13. caller with no viewer projects gets an empty collection (not an error).

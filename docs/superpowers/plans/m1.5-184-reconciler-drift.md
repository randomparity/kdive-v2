# Plan — #184 reconciler/teardown drift-repair validation (M1.5 issue 5)

ADR-0074 (engine→port wiring) · ADR-0072 (seeded engine) · ADR-0021 (reconciler passes) ·
Spec `m1.5-fault-injection-provider.md` §Validation surface · builds on #182 (engine on main).

## Goal

Drive each of the five existing reconciler drift-repair passes with a **real, seed-pinned
fault** (or the inventory seam) and assert the **exact** post-repair state. The reconciler
passes already exist and are tested with hand-seeded rows; this issue's distinct value is
proving the **fault engine** (and the mock infra-inventory) produce the drift the reconciler
repairs. The only production code added is the thin **faulting wrapper** (ADR-0074) that lets
a seeded `FaultEngine` perturb the fault-inject provision/install ports. No new MCP tools, no
new `ErrorCategory`, no migration.

## Key facts established by reading the code

- **The engine is pure/unwired** (`providers/fault_inject/engine.py`): `FaultEngine.decide(
  system_id, plane, attempt) -> FaultDecision(fail, category, latency_s)`. `category` is
  non-None iff `fail`. `from_capabilities` builds it from the resource `capabilities` jsonb
  (`seed`/`fault_rate`/`max_latency_s`). No provider port consults it today.
- **Happy-path ports** (`providers/fault_inject/provider.py`) are synthetic and draw no
  faults: `FaultInjectProvision.provision` mints `fault-inject-<system_id>` and records it in
  the shared `FaultInjectInventory`; `FaultInjectInstall.install/boot` are no-ops.
- **The reconciler passes** (`reconciler/loop.py`), with their *exact* triggers:
  - `_repair_orphaned_systems` (321-359): System state **NOT** in `(TORN_DOWN, FAILED)` AND
    its Allocation **in** `(RELEASED, EXPIRED, FAILED)` → enqueues a `teardown` job keyed
    `<system_id>:teardown`, principal `system:reconciler`. Counts only a *new* enqueue.
  - `_repair_abandoned_jobs` (362-419): job `running`, `lease_expires_at < now()`,
    `attempt >= max_attempts` → job `failed`/`error_category=lease_expired`; if the payload
    carries a `run_id` (BUILD/INSTALL/BOOT only) whose Run is in `(CREATED, RUNNING)`, that
    Run → `failed`/`failure_category=lease_expired`. **PROVISION carries a `system_id`, not a
    `run_id`** — so the run-compensation path needs a run-bearing job (INSTALL/BOOT).
  - `_repair_dead_sessions` (422-439): `live` session, `worker_heartbeat_at` non-NULL and
    `< now() - stale_after` → `detached`. Triggers on the **stale heartbeat**, not a fault
    category.
  - `_repair_leaked_domains` (442-491): for each `reaper.list_owned()` domain with a non-None
    `system_id`, if no live `systems` row (absent or `torn_down`) AND no in-flight teardown →
    `reaper.destroy(name)`. Driven by the **inventory seam**, no engine draw.
- **`provision_handler`** (`planes/systems.py:219-223`): a `CategorizedError` from the
  provisioner → `_record_provision_failure` → `System → FAILED`, then re-raise.
- **`install_handler`/`boot_handler`** (`planes/runs.py`): a `CategorizedError` →
  `_abandon_run_step_best_effort` + **re-raise**; they do **not** transition the Run. The Run
  is failed only downstream (worker `queue.fail` dead-letters the job; the reconciler
  compensates the Run).
- **Test conventions** (`tests/reconciler/conftest.py`): disposable Postgres via
  `migrated_url`; seed on an autocommit connection; run repairs through a real non-autocommit
  pool (`run_repair`). `seed_system`/`seed_run`/`seed_debug_session`/`seed_running_job` exist.
  `FakeReaper`/`_FakeDomain` duck-type the `InfraReaper`/`OwnedDomain` ports. The existing
  lease tests seed an already-lapsed lease (`lease_seconds=-60`) rather than waiting.
- **The integration spine test** (`tests/integration/test_fault_inject_spine.py`) drives
  `provision_handler` directly with `build_provider_resolver(enable_fault_inject=True)` against
  disposable Postgres (ADR-0019: drive handlers, not transport).
- **`ResourceKind.FAULT_INJECT`** exists; `FAULT_INJECT` is admitted by migration `0018`.

## The faulting wrapper (the only production code added)

New module `src/kdive/providers/fault_inject/faulted.py`:

- `FaultedProvision(inner: FaultInjectProvision, engine: FaultEngine, *, attempt_for:
  Callable[[UUID], int] = lambda _sid: 1, sleep: Callable[[float], Awaitable[None]] =
  asyncio.sleep)`.
  - `provision(system_id, profile)` is **async-aware via the existing `to_thread` seam**:
    the port protocol is sync (`Provisioner.provision` is called via `asyncio.to_thread` in
    the handler). The wrapper therefore keeps a **sync** `provision` that (a) computes
    `decision = engine.decide(system_id=system_id, plane=PROVISION, attempt=attempt_for(
    system_id))`, (b) if `decision.latency_s > 0`, blocks for it via a **sync sleep seam**
    (`sleep_s: Callable[[float], None] = time.sleep`) — because the handler already offloads
    the whole call to a thread, a blocking sleep there does not stall the event loop, matching
    how a real slow provider behaves; (c) if `decision.fail`, raises
    `CategorizedError(decision.category)` (guaranteed non-None); (d) else delegates to
    `inner.provision`. `teardown`/`reprovision` delegate unchanged.
  - The provider ports are sync (called via `to_thread`), so the injected seam is a **sync**
    `sleep_s` (default `time.sleep`), per ADR-0074: inject the sleep so tests don't wall-block,
    and assert the engine-computed `latency_s`.
- `FaultedInstall(inner: FaultInjectInstall, engine: FaultEngine, *, attempt_for, sleep_s)`:
  same shape over `install`/`boot` with `plane=INSTALL`/`plane=BOOT`.
- The wrappers hold **no counter**: `attempt` comes from `attempt_for` (default constant 1),
  honoring ADR-0072's "attempt from durable state, never a process-local counter."

`build_faultinject_runtime` gains an optional `engine: FaultEngine | None = None`; when given,
it wraps the provision/install/boot ports in the faulting wrappers. When `None` (the default,
the happy path), composition is unchanged — the happy-path spine test still passes untouched.

## TDD task list (one test+impl per logical step; guardrails green at each commit)

Order is chosen so the wrapper exists before the tests that consume it; the inventory-seam
case has no wrapper dependency and can land first.

1. **Wrapper unit tests + impl.** `tests/providers/fault_inject/test_faulted.py`:
   - a seed/plane pinned to `fail` raises `CategorizedError` with the catalog category;
   - a seed pinned to no-fail delegates and returns the synthetic domain;
   - `latency_s > 0` calls the injected `sleep_s` with the engine's computed delay and the
     delay equals `decide(...).latency_s` (assert against the real engine value);
   - `attempt_for` is threaded into the draw (two `attempt_for` values give different draws);
   - the wrapper never sleeps/raises when the plane's `fault_rate`/`max_latency_s` is absent.
   Then write `faulted.py`. Commit.

2. **Leaked-domain via the mock inventory seam.** New
   `tests/reconciler/test_drift_fault_inject.py::test_leaked_fault_inject_domain_is_reaped`:
   build a `FaultInjectInventory`, `record(system_id, "fault-inject-<sid>")` with **no**
   `systems` row, wrap it in `FaultInjectReaper`, run `_repair_leaked_domains` via `run_repair`
   → assert count 1 and the domain is `forget`-ten (gone from `owned_domains()`). This proves
   the real mock seam (not `FakeReaper`) satisfies the `InfraReaper` port and the reaper reaps
   it. No engine. Commit.

3. **Orphaned-System after a real fault-inject provision.** In the same drift test module:
   seed a fault-inject Resource + **terminal** Allocation (`RELEASED`) + a System driven to a
   live non-terminal state by a **successful** wrapper provision (seed pinned to no-fail; the
   wrapper delegates and records the domain), then run `_repair_orphaned_systems` → assert a
   `teardown` job enqueued with `system:reconciler`. Drive the provision through
   `provision_handler` with the faulted resolver so the System reaches `ready` via the real
   handler (mirrors the spine test), then release the allocation, then reconcile. Commit.

4. **Lease-expiry-mid-job → Run failed(lease_expired), distinct from canceled.** Seed a
   run-bearing (INSTALL) job with `attempt >= max_attempts` and an **already-lapsed** lease
   (`lease_seconds=-60`) carrying a `run_id` whose Run is `RUNNING`. **Independently**, build
   the engine from a `capabilities` map whose `max_latency_s[install]` is large and assert
   `engine.decide(..., plane=INSTALL).latency_s > <the short lease window>` — proving the
   latency lever *would* outlast the lease (the seed-derived magnitude is real). Then run
   `_repair_abandoned_jobs` → assert the job is `failed`/`lease_expired` AND the Run is
   `failed`/`failure_category=lease_expired` AND `RunState != canceled`. Commit.

5. **Abandoned-job (worker death) shares the lease-expiry sweep.** A focused test that a
   run-bearing zombie with a lapsed lease + exhausted attempts is dead-lettered and its Run
   compensated — same `_repair_abandoned_jobs`, framed as worker-death (no live latency). If
   step 4 already covers the run-compensation assertion, this step asserts the *job*-side
   `abandoned`/dead-letter state distinctly (job `failed`/`lease_expired`) to keep the two
   issue bullets each have an explicit assertion. Commit.

6. **Dead-DebugSession from a connect transport-drop.** Assert the engine draws a
   `connect`/`TRANSPORT_FAILURE` fail for a pinned seed (the upstream cause), then seed a
   `live` DebugSession with a **stale** `worker_heartbeat_at` (the state the dropped transport
   produces — the worker stopped beating) and run `_repair_dead_sessions` → assert `detached`.
   The two halves (engine draws the transport fault; reconciler detaches on staleness) are
   asserted together to show cause→trigger, per ADR-0074. Commit.

7. **Wire the engine into `build_faultinject_runtime` (optional `engine` arg) + a guard
   test.** Test: `build_faultinject_runtime(engine=FaultEngine(...))` returns a runtime whose
   `provisioner` is a `FaultedProvision`; with `engine=None` it is the bare
   `FaultInjectProvision` (happy path unchanged). Confirm the existing
   `tests/integration/test_fault_inject_spine.py` and `tests/providers/test_composition.py`
   still pass (no behavior change when `engine` is omitted). Commit.

## Edge / error paths each test must cover (not just the happy case)

- Wrapper: `fail=True, category` is always non-None (assert the wrapper never passes `None`);
  absent-plane config → no sleep, no raise, pure delegation.
- Orphaned-System: a System left `failed` by a provision `fail` draw is **NOT** orphan-reaped
  (negative test — proves the `fail` draw is the wrong trigger, locking ADR-0074's decoupling).
- Lease-expiry: assert the Run is `failed` with category `lease_expired`, explicitly **not**
  `canceled` (the issue's distinct-from-canceled requirement) and **not** a catalog category.
- Dead-session: a NULL-heartbeat live session is **not** detached (the dropped-transport cause
  must have lapsed the heartbeat first) — reuse the existing NULL-heartbeat invariant.
- Leaked-domain: a domain whose `systems` row is still live is **not** reaped (mock-seam
  parity with the existing `FakeReaper` live-row test).

# M1.2 ‚Äî Live-Stack End-to-End Validation (Integration Contract)

## Purpose

M1.2 is the first **executable, real-infrastructure** end-to-end test of kdive. The M0
walking-skeleton full-path test was wired but left a `NotImplementedError` stub, and no
test drives the MCP tools over their actual transport. M1.2 closes both gaps: it drives
the full spine ‚Äî allocate ‚Üí provision ‚Üí build ‚Üí install ‚Üí boot ‚Üí attach ‚Üí force-crash ‚Üí
capture vmcore ‚Üí release ‚Äî **over the live MCP HTTP stack** (server + worker + reconciler
against containerized Postgres + S3/MinIO + mock-OIDC), under real OIDC role tokens, on a
host with real libvirt. It is operator-run, not GitHub-CI.

  (the umbrella spec; sub-issues A‚ÄìF)

  full-path tier of [ADR-0035](../adr/0035-walking-skeleton-e2e-harness.md))
- **Parent:** [`top-level-design.md`](top-level-design.md) ¬ßRoadmap (M1.2)
- **Depends on:** **M1.1** (platform-RBAC) ‚Äî the `report` phase exercises

## What M1.2 adds to M1.1

- A reusable **MCP-over-HTTP test harness** (`fastmcp.Client`) and an **OIDC
  token-issuance helper** that mints per-project `viewer`/`operator`/`admin` tokens **and a
  `platform_auditor` token** from the mock issuer (exercising the real JWKS/`JWTVerifier`
  path, not the in-process `mint`).
- A **phase-structured spine driver** (`tests/integration/test_live_stack.py`, new
  `live_stack` marker) that drives every step as a tool call over HTTP, against the
  shipping `server` + `worker` + `reconciler` processes ‚Äî **replacing** (deleting)
  `test_walking_skeleton_full_path`.
- **Stack orchestration**: `just stack-up` / `just test-live-stack`, host
  `server`/`worker`/`reconciler` wired to the existing compose backends, plus a runbook.

## Non-goals

- **No new product code.** The driver exercises shipped handlers; the one tool it needs
  beyond M0/M1, `accounting.report`, is built in **M1.1** and consumed here.
- **No GitHub-CI integration** ‚Äî operator-run on a KVM host; `live_stack` is deselected on
  `pull_request` exactly as `live_vm` is.
- **No containerized kdive service** ‚Äî `server`/`worker`/`reconciler` run on the host
  against containerized backends; containerizing them is a **deferred follow-on**
  (sub-issue F), not part of M1.2's first pass.
- **No handler/behaviour changes** to existing planes; reuses ADR-0035's
  `scripts/live-vm/*` fixtures and preflight-skip idiom.

## Schema / tool surface

No schema delta. No new tools (drives existing M0/M1 tools + M1.1's `accounting.report`
over the wire). Test-surface delta only: new `live_stack` pytest marker; the
`test_walking_skeleton_full_path` stub is deleted.

## Exit criteria

Falsifiable; asserted by the spine driver on a KVM host with fixtures + a running stack.
A failure names its **phase**.

1. **Protocol.** Every step returns a well-formed MCP envelope **over HTTP**, and tokens
   validate through the real JWKS/`JWTVerifier` path.
2. **#1 path completes.** A fetchable redacted vmcore artifact exists in MinIO at the end.
3. **#2 attribution.** An `audit_log` row for every transition + the `force_crash`, under
   the request's `(principal, agent_session, project)` tuple.
4. **#5 teardown.** After release the System is `torn_down` **and**
   `Discovery.list_owned()` returns no `OwnedInfra` for the released `system_id`.
5. **RBAC negatives** (over the wire). A `viewer` token is denied operator/admin
   operations; `force_crash` is denied without the `admin` role.
6. **#3 redaction.** A planted secret does not leak through the wire path (transcript +
   artifact-sensitivity).
7. **Report phase.** `accounting.report`'s all-projects form (M1.1) is reachable under a
   `platform_auditor` token and returns this run's spend; that form is denied to a
   project-only token. (Multi-project rollup correctness, and the membership-gated
   granted-set form, are M1.1's in-process tests, not here ‚Äî the spine runs one project.)
